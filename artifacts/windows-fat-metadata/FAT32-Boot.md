# FAT32 Boot Sector

## Forensic value
The authoritative source of a FAT32 volume's identity. Carries the 4-byte volume serial number (VSN) at offset 0x43 that every FAT32-scoped shell artifact (LNK DriveSerialNumber, ShellBag drive-node serial, jump-list embedded LNK) records at access time. For USB flash storage — the dominant FAT32 deployment today — this sector is the pivot back from per-user artifacts to the physical volume.

C3 ceiling: the sector is on-disk and persists, but admin users can edit raw sectors, and `format.exe` rewrites VSN trivially. No signature or integrity check.

## Cross-references
- **FilesystemVolumeSerial** concept — offset 0x43 is the authoritative source the shell artifacts record.
- **PartitionDiagnostic-1006** — the kernel captures the raw FAT32 VBR in `Vbr0/Vbr1/Vbr2` fields at mount time. Allows historical VSN recovery for reformatted volumes where only the EVTX retains the pre-format serial.
- **ShellLNK, ShellBags, Recent-LNK, jump lists** — consumers of the VSN this sector produces.

## Known quirks
- **Two volume labels**. FAT32 stores one in the boot sector (offset 0x47) and a separate one as a VOLUME_ID directory entry in the root. Live rename usually updates only the directory entry, leaving the boot-sector copy stale. The discrepancy is informational but not a tamper signal — it's the common case.
- **OEM-ID is NOT a filesystem identifier**. Use the BPB layout + bootstrap magic, not the OEM-ID string, to identify FAT32. Custom formatters write arbitrary OEM-ID strings.
- **Backup boot sector at relative sector 6** normally holds a byte-identical copy. Discrepancy between sector 0 and sector 6 is a high-confidence tamper signal — `format.exe` updates both; a hand hex-edit often misses the backup.

## Practice hint
- Format a USB as FAT32 (`format X: /FS:FAT32 /V:TESTVOL /Q`).
- Dump sector 0: `dd if=\\.\X: bs=512 count=1 | xxd | grep -E "^0000004[0-9a-f]"`. The 4 bytes immediately after offset 0x43 are the VSN (little-endian).
- Open a file on the USB from a Windows user account. Parse the resulting `%USERPROFILE%\AppData\Roaming\Microsoft\Windows\Recent\*.lnk` with `LECmd.exe`. The `DriveSerialNumber` field equals the VSN you read from the boot sector.
- Check sector 6 for the backup copy: `dd if=\\.\X: bs=512 skip=6 count=1 | xxd`. Compare. They should be identical.
