# MBR Disk Signature

## What it is
A 4-byte random value written by Windows to offset 0x1B8 of a disk's MBR at disk initialization. Windows uses it as the unique key for the physical disk in:
- the **Boot Configuration Data** store (MBR-partitioned boot volumes reference their disk by signature)
- the **MountedDevices** registry (volumes on MBR disks are bound via signature + byte-offset of the partition)
- **kernel mount events** (Partition/Diagnostic-1006 EventData\Mbr\Signature)

Generated by `diskmgmt.msc`, `diskpart`, or Windows Setup on first disk initialization. Once written, Windows will preserve it indefinitely unless explicitly cleared (`diskpart clean`) or reinitialized.

## How it differs from adjacent identifiers

| Identifier | Scope | Changes when | Sourced from |
|---|---|---|---|
| **MBRDiskSignature** | physical disk (whole-disk) | MBR is rewritten (diskpart clean, Setup, vendor recovery) | MBR sector 0 offset 0x1B8 |
| **FilesystemVolumeSerial** | volume (partition-scoped) | filesystem is reformatted | NTFS $Boot 0x48, FAT32 0x43, exFAT 0x64 |
| **VolumeGUID** | volume (mount-manager-assigned) | volume is first mounted (random) | MountedDevices, MountPoints2 |
| **DeviceSerial** | USB device (hardware) | never (firmware-level); or when OS-synthesized, per-device-instance | USB descriptor, USBSTOR, Partition/Diagnostic-1006 |
| **GPT Disk GUID** | physical disk (GPT equivalent) | MBR→GPT conversion, diskpart clean | GPT header (LBA 1) — not yet an authored artifact |

## Encoding variations

| Artifact | Where | Encoding |
|---|---|---|
| MBR | offset 0x1B8 of sector 0 | uint32 LE |
| MountedDevices | binding-data Format 2 (12 bytes) — first 4 bytes | uint32 LE |
| PartitionDiagnostic-1006 | EventData\\Mbr\\Signature | uint32 (XML integer text) |

All three are the same 4-byte value. Cross-correlation between them is direct-equality comparison (after endian normalization).

## Forensic value
- **Device-persistence pivot below the filesystem**. Survives reformat, which FilesystemVolumeSerial does not. A disk with a stable MBRDiskSignature seen across time — with varying FilesystemVolumeSerial values — is a reformat timeline for that disk.
- **MountedDevices disambiguation**. When binding-data is 12 bytes (Format 2 — MBR), the signature names the disk AND the 8-byte offset names the partition within it. Together: `{signature, partition-offset}` uniquely identifies a partition on an MBR disk.
- **Disk-collision detection**. Two disks with the same signature on the same system is an anomalous state — Windows auto-regenerates one signature and logs the change (System.evtx, PartMgr source). Historical signature-collision investigations leverage Partition/Diagnostic-1006 records.

## Limitations
- **Not present on GPT disks**. GPT's protective MBR typically has zeros at offset 0x1B8. For GPT, use the 16-byte disk GUID from the GPT header. Authored as a separate concept when GPT-Header artifact is added.
- **Admin-writable with no integrity mechanism**. `diskpart uniqueid disk id=<hex>` sets it directly. Trivially tamperable — but tamper is detectable by comparing to kernel-recorded Partition/Diagnostic-1006 historical records.
- **Zero-signature ambiguity**. `0x00000000` means either "never initialized" or "post-`diskpart clean`." Adjacent artifact context (Partition/Diagnostic history, MountedDevices binding presence) resolves which.
